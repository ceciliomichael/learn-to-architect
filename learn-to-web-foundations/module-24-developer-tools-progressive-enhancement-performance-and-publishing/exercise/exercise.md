# Exercise: Developer Tools, Progressive Enhancement, Performance, and Publishing

## Goal

Apply this module in a page whose structure, styling, behavior, and accessibility can be inspected.

## Steps

1. Combine semantic structure, navigation, responsive CSS, and accessible focus behavior into a small site.
2. Test the site with CSS disabled and images unavailable.
3. Use developer tools to resolve every failed request and unexplained console message.
4. Measure the largest resources and remove unnecessary bytes.
5. Publish a clean copy over HTTPS and retest paths, forms, keyboard use, zoom, and narrow layouts.

## Success check

Every step has observable browser evidence. The result remains usable with keyboard input, text zoom, narrow space, and unavailable optional resources whenever those conditions apply.
