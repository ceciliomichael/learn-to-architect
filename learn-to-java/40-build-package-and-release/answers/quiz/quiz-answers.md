# Quiz answers

1. Release from a clean checkout with locked or reviewed dependency resolution.
2. Inspect the JAR contents and launch it in an environment like production.
3. jlink can create a custom runtime and jpackage can create native installers, each with platform requirements.
4. Record source commit, JDK, build command, dependencies, target, checksums, and signing process.
5. Keep a previous known-good artifact and write rollback verification before rollout.
