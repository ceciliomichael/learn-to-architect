# Quiz answers

1. Document caller obligations, results, failures, thread rules, units, and examples.
2. Public signatures, exceptions, formats, service behavior, and dependency requirements can all break callers.
3. Deprecation needs a reason, supported replacement, and migration period.
4. Review source, ownership, license, advisories, transitive graph, plugins, and update history.
5. Update in focused changes, run compatibility tests, inspect resolved diffs, and keep rollback possible.
