# Module 01: Create a Next.js Application and Read the Generated Project

## Before you start

Complete the React course first. You should already understand components, props, state, effects, forms, and TypeScript.

For local work, install an actively supported LTS release of Node.js. This course uses Node.js 24. After installation, open a new terminal and check both programs:

```text
node --version
npm --version
```

The first command should start with `v24.`. The second should print an npm version. If either command is not found, reinstall Node.js from its official website and reopen the terminal.

## What you will learn

By the end of this module, you will be able to:

- create a TypeScript Next.js project with the official generator
- explain what the generator adds to the project
- start and stop the development server
- find the App Router entry files
- distinguish source files, dependencies, scripts, and generated output

## 1. Create a safe practice project

Move to a folder where you keep practice projects. Do not run the generator inside this course repository.

Run:

```text
npx create-next-app@latest nextjs-course --use-npm
```

`npx` downloads and runs the official `create-next-app` package. `@latest` asks npm for the current stable generator. `nextjs-course` becomes the new project folder. `--use-npm` keeps this course on one package manager.

Choose these answers when prompted:

```text
Use the recommended Next.js defaults? No, customize settings
Use TypeScript? Yes
Which linter? ESLint
Use React Compiler? No
Use Tailwind CSS? No
Put code inside a src directory? No
Use App Router? Yes
Customize the import alias? No
```

The course introduces CSS and optimization separately. React Compiler is also left off until you understand ordinary React rendering. Choosing fewer features now keeps the generated project readable.

## 2. Enter the project and start it

```text
cd nextjs-course
npm run dev
```

Keep that terminal open. Visit `http://localhost:3000` in a browser. A starter page should appear.

Press `Ctrl+C` in the terminal when you want to stop the server. A server occupies the terminal until it is stopped.

## 3. Read the important files

Your project contains files similar to these:

```text
nextjs-course/
  app/
    favicon.ico
    globals.css
    layout.tsx
    page.tsx
  public/
  eslint.config.mjs
  next-env.d.ts
  next.config.ts
  package-lock.json
  package.json
  tsconfig.json
```

- `app/page.tsx` defines the page shown at `/`.
- `app/layout.tsx` wraps every page below it. The root layout owns the `html` and `body` elements.
- `app/globals.css` contains styles available throughout the application.
- `public` stores files served directly, such as images.
- `package.json` records dependencies and reusable commands.
- `package-lock.json` records the exact dependency tree installed by npm. Commit it with the project.
- `tsconfig.json` contains TypeScript settings.
- `next.config.ts` contains optional Next.js configuration.
- `.next` appears after running or building the project. It is generated output and should not be edited.
- `node_modules` contains installed packages. It is generated by `npm install` and should not be edited or committed.

## 4. Read the project scripts

Open `package.json` and find `scripts`. A generated project normally includes commands with these responsibilities:

- `npm run dev` starts the development server.
- `npm run build` creates and checks a production build.
- `npm start` runs an already-built production application.
- `npm run lint` checks source files with the configured linter.

The exact implementation behind a script may change between Next.js releases. Use the script name instead of copying internal commands from old tutorials.

## 5. Replace the home page

Replace `app/page.tsx` with:

```tsx
export default function HomePage() {
  return (
    <main>
      <h1>Study Planner</h1>
      <p>Plan one focused learning session at a time.</p>
    </main>
  );
}
```

Save the file while the development server is running. The browser should update without restarting the server.

The function is a React Server Component by default. Do not add `"use client"`; this page does not use browser-only state or event handlers.

## 6. Verify more than the screen

Run these checks after stopping the development server:

```text
npm run lint
npm run build
```

A page appearing in the browser proves only the development path. The production build also checks compilation, routes, and server rendering work together.

## Common mistakes

### Running the command in the wrong folder

The generator creates a new folder. Check your current location before running it so you do not mix projects.

### Following Pages Router instructions

This course uses the App Router. Files such as `pages/index.tsx` and `_app.tsx` belong to the older Pages Router structure and are not used here.

### Editing generated folders

Do not edit `.next` or `node_modules`. Change source or configuration files and let the tools regenerate their output.

### Using a type cast to silence uncertainty

Strict TypeScript helps only when its warnings are investigated. Do not add `as any` merely to make an error disappear.
