# Exercise: Package and Release C Programs

## Steps

1. Run normal, version, and invalid argument paths.
2. Write prerequisite, build, test, and run instructions.
3. Archive source, build files, README, license, and notes.
4. Unpack and rebuild in an empty directory.
5. Record a checksum and check for secrets or temporary files.

## Success check

The program is identifiable, and a clean recipient can reproduce the documented build from the package alone.
