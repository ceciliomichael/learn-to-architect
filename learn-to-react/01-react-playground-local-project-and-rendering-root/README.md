# Module 01: React Playground, Local Project, and Rendering Root

## Before you start

Complete Frontend Development with TypeScript first. This module assumes you can use HTML, browser modules, DOM elements, npm packages, and strict TypeScript.

Install Node.js 24 LTS for local work. Open a new terminal and verify the installation:

```text
node --version
npm --version
```

The Node.js result should start with `v24.`. If either command is missing, install Node.js from its official website and reopen the terminal.

## What you will learn

By the end of this module, you will be able to:

- try a component without installing anything
- create a local React and TypeScript project with Vite
- explain the roles of `index.html`, `src/main.tsx`, and `src/App.tsx`
- render one React application into one DOM element
- run development, type-checking, and production-build commands

## 1. Try React online or work locally

An official React documentation sandbox is useful for a quick experiment because it already provides the compiler and packages. A local project is better for learning files, dependencies, scripts, tests, and builds. Complete the local setup even if you try the sandbox first.

Create the local project in a folder used for disposable practice:

```text
npm create vite@latest react-course -- --template react-ts
cd react-course
npm install
npm run dev
```

The extra `--` passes `--template react-ts` through npm to Vite. The template installs React source files written in TypeScript. Vite prints a local address, commonly `http://localhost:5173`. Open the exact address shown in your terminal.

Create React App is deprecated. Do not use `create-react-app` for this course.

## 2. Read the generated project

The important starting files are:

```text
react-course/
  public/
  src/
    App.css
    App.tsx
    index.css
    main.tsx
  eslint.config.js
  index.html
  package-lock.json
  package.json
  tsconfig.app.json
  tsconfig.json
  tsconfig.node.json
  vite.config.ts
```

- `index.html` contains the DOM element where React is mounted.
- `src/main.tsx` finds that element and creates the React root.
- `src/App.tsx` defines the first component rendered inside the root.
- `package.json` lists dependencies and scripts.
- `vite.config.ts` configures the development and build tool.
- `tsconfig.app.json` contains TypeScript rules for browser source code.
- `node_modules` contains installed packages and should not be edited or committed.
- `dist` is generated by a production build and should not be edited.

## 3. Follow the rendering path

Open `index.html` and find:

```html
<div id="root"></div>
```

React does not replace the whole browser by magic. `src/main.tsx` finds this real DOM element and gives it to React:

```tsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./App";

const container = document.getElementById("root");

if (container === null) {
  throw new Error("Expected an element with id root");
}

createRoot(container).render(
  <StrictMode>
    <App />
  </StrictMode>,
);
```

`document.getElementById` can return `null`, so the code checks before calling `createRoot`. TypeScript then knows `container` is an element. `StrictMode` enables additional development checks. It does not render a visible HTML element.

## 4. Write the first component

Replace `src/App.tsx` with:

```tsx
export default function App() {
  return (
    <main>
      <h1>Study Planner</h1>
      <p>My first React component is rendering.</p>
    </main>
  );
}
```

A component is a function whose name starts with a capital letter and which returns React elements. `<App />` asks React to call that component and describe its output.

Save the file. The page should update while the development server remains running.

## 5. Check the project

Stop the server with `Ctrl+C`, then run:

```text
npm run lint
npm run build
```

The generated build script performs TypeScript checking before creating `dist`. Restart `npm run dev` when you want to continue editing.

## Common mistakes

### Skipping `npm install`

The generator writes dependency names to `package.json`. `npm install` downloads them into `node_modules`.

### Removing the null check

The root element comes from HTML at runtime. A TypeScript type cannot guarantee that the element was actually written in `index.html`.

### Mounting several unrelated roots

Most applications have one React root. Use components to divide the interface instead of creating a root for every section.

### Editing generated output

Change files in `src`, not files inside `node_modules` or `dist`.
