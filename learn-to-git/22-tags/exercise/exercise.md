# Exercises: Mark Releases with Tags

Use the local `pick-practice` repository.

## Exercise 1: Create and inspect tags

1. Confirm the current branch and clean state.
2. Create annotated `v1.0.0` at `HEAD` with a release message.
3. Create lightweight `practice-point` at `HEAD^`.
4. List tags and inspect each one.
5. Show the decorated graph.

## Exercise 2: Prove tags stay put

1. Add and commit a small `after-release.txt` file.
2. Show the graph.
3. Confirm `main` moved but `v1.0.0` did not.
4. Compare `v1.0.0` with `HEAD`.

## Exercise 3: Correct a private tag

1. Delete `practice-point`.
2. Confirm it is gone and `v1.0.0` remains.
3. Do not publish either tag in this exercise.
