# Exercises: Create Your First Commits

Use the `three-locations` repository.

## Exercise 1: Commit the staged file

1. Inspect status.
2. Commit the staged `notes.txt` with the message `Add first Git notes`.
3. Inspect status again.
4. Show the new commit in one-line form with file statistics.

## Exercise 2: Keep an unstaged edit out

1. Add `Commits should have a clear purpose.` to `notes.txt`.
2. Create `later.txt` containing `Do not commit this yet.`
3. Stage only `notes.txt`.
4. Inspect status and predict what the commit will include.
5. Commit with `Explain focused commits`.
6. Inspect status and prove that `later.txt` was not committed.

## Exercise 3: Make the second file meaningful

1. Replace the text in `later.txt` with `Review status before every commit.`
2. Stage and commit it with a clear message of your own.
3. Show the newest commit.
