# Exercises: Inspect Changes with Diff

Use the repository from Module 03.

## Exercise 1: Follow a change

1. Add `Diff shows changed lines.` to `notes.txt`.
2. Run status and plain diff.
3. Stage the file.
4. Run plain diff again, then staged diff.
5. Explain why one view is now empty.

## Exercise 2: Create two versions at once

1. Keep the staged change from Exercise 1.
2. Add `A later edit stays in the working tree.` to the file without staging it.
3. Inspect status, plain diff, and staged diff.
4. State which line each diff contains.

## Exercise 3: Finish carefully

1. Stage the latest file.
2. Review the staged diff for only `notes.txt`.
3. Commit with `Explain diff views`.
4. Compare the new commit with its parent.
