# Exercises: Profiling, Optimization, and Releasing Go Programs

1. Benchmark one representative operation with allocations reported.
2. Capture a CPU profile from a bounded test workload and inspect cumulative time.
3. Improve one measured bottleneck while keeping tests green.
4. Build a release artifact and inspect it with `go version -m`.
5. Record source revision, Go version, target, checksum, verification steps, and rollback approach.
