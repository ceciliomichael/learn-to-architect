# Web Development Learning Path

This is the starting point for a complete beginner who wants to become a web developer. The path separates lasting web knowledge from framework-specific knowledge, so changing tools later does not mean starting over.

## Required shared path

1. **Web Foundations:** Learn how the web works, build semantic HTML documents, style responsive layouts, create usable forms, and include accessibility from the beginning.
2. **Learn TypeScript:** Learn programming, JavaScript runtime behavior, modules, asynchronous work, runtime validation, installed packages, testing, and maintainable type design.
3. **Frontend Development with TypeScript:** Use TypeScript with the DOM, events, forms, HTTP, storage, browser security, testing, accessibility, performance, and deployment without a framework.

## Choose a frontend specialization

After the shared path, choose one primary framework. You do not need to complete both.

### React path

1. **React:** Learn components, props, state, forms, effects, hooks, testing, and performance.
2. **Next.js:** Continue into routing, server and client components, server data, caching, mutations, authentication, databases, and deployment.

### Vue path

1. **Vue:** Learn templates, reactivity, components, composables, Router, Pinia, testing, and deployment.
2. A future Nuxt specialization may follow Vue, but it is not required by the current path.

## Backend choices after the shared path

Frontend learners can later choose a server specialization using TypeScript, Go, Java, Python, or Rust. The existing SQL course provides the relational database foundation. The language courses remain independent references; they do not replace the shared lessons about HTTP, browser security, accessibility, or user experience.
