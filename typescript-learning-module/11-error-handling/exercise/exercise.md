# Welcome to Practice Lab: Error Handling

Welcome to the interactive exercise lab for **Error Handling**! Here is where theoretical knowledge transforms into practical coding ability.

In production engineering, robust error handling is what distinguishes fragile applications from resilient, fault-tolerant enterprise systems. The TypeScript compiler is your mentor: experiment with type narrowing, custom error hierarchies, and functional failure handling.

## What You Will Tackle in This Lab

1. **[Challenge 01: try / catch / finally in Practice](./challenge-01.md)** - Master JavaScript's exception handling blocks, type narrowing on `unknown` catch variables, and guaranteed resource cleanup in `finally`.
2. **[Challenge 02: Custom Error Classes](./challenge-02.md)** - Build an enterprise domain error hierarchy with prototype restoration (`Object.setPrototypeOf`), custom identifiers, and precise `instanceof` branching.
3. **[Challenge 03: The Result Pattern](./challenge-03.md)** - Implement functional programming error handling using Discriminated Unions (`Result<T, E>`) to eliminate runtime exception throws entirely.

## How to Work on These Challenges

1. Open each challenge file sequentially in your IDE.
2. Read the architectural scenario and technical requirements carefully.
3. Write your TypeScript code directly inside the **ANSWER HERE** section.
4. If you need conceptual reinforcement, review the theoretical deep dives in **[../README.md](../README.md)**.
