# Welcome to Module 01 Conceptual Quiz: The Basics

Writing code is only half the battle of becoming a great software engineer. Senior engineers must also understand **how tools work under the hood**. 

This conceptual quiz is designed to test your mental models of the TypeScript compiler (`tsc`), variable immutability, and runtime JavaScript behavior. These exact types of questions appear regularly in technical job interviews.

---

## Topics We Are Testing Today

1. **[Question 01: Type Erasure & Output](./question-01.md)**: What happens to your type annotations when TypeScript compiles your code into JavaScript?
2. **[Question 02: Array Protection & `const`](./question-02.md)**: Does `const` prevent you from pushing new items into an array? Let us analyze how variable references work.
3. **[Question 03: Implicit Type Inference](./question-03.md)**: Why does TypeScript catch errors even when you do not write `: string` or `: number`?
4. **[Question 04: `any` vs `unknown`](./question-04.md)**: Why is one considered dangerous and the other considered professional practice?
5. **[Question 05: String & Array Transformations](./question-05.md)**: Practice manipulating raw string data safely into typed arrays.

---

## How to Take This Quiz

1. Open each question file sequentially.
2. Read the prompt and think about what happens inside the JavaScript engine and compiler.
3. Write your explanation or record your answer in the **`ANSWER HERE`** section.
4. Try explaining *why* your answer is correct in your own words. If you can explain it clearly without looking at notes, you have truly mastered the concept.
