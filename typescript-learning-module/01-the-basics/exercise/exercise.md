# Welcome to Module 01 Practice Lab: The Basics

Congratulations on completing the theory section! Now it is time to put your hands on the keyboard and build real muscle memory. 

Learning TypeScript is just like learning a musical instrument: you cannot just read about chords; you have to play them! Do not worry if you make mistakes or if the compiler yells at you. **Red squiggly lines are your best friend here.** Every error you solve makes you a stronger engineer.

---

## What You Will Tackle in This Lab

In this first lab, we are focusing on core syntax and compiler habits:
1. **[Challenge 01: Fix the Broken Code](./challenge-01.md)**: Step into the shoes of a code reviewer! You will inspect a buggy script written by a junior developer, identify type mismatches, and explain why the compiler rejected them.
2. **[Challenge 02: Build a Typed Inventory](./challenge-02.md)**: Build strict array structures for an RPG game inventory using primitive types (`string[]`, `number[]`, `boolean[]`) and array methods.
3. **[Challenge 03: Safe Unknown Input](./challenge-03.md)**: Practice handling dangerous external data! You will use `unknown` and runtime `typeof` guards to safely inspect variables without bypassing type safety.

---

## How to Work on These Challenges

1. Open a challenge file (for example: `challenge-01.md`).
2. Read the scenario and instructions carefully.
3. Scroll down to the **`ANSWER HERE`** code block.
4. Write your TypeScript implementation directly inside the block.
5. *Tip:* If you have VS Code open, create a temporary `.ts` scratch file to test your code and watch the compiler provide real-time feedback.
