Now that you have practiced defining data blueprints, let us test how well you understand TypeScript's architectural philosophy.

3. **[Question 03: Structural Typing vs Nominal Typing](./question-03.md)**: How does TypeScript evaluate compatibility across object literals, interfaces, and classes? If a `Dolphin` class has the same properties as a `Swimmer` interface, can you pass a `Dolphin` where a `Swimmer` is required?
