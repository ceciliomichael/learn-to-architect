# Module 01: Vue Playground, create-vue, and the Application Root

## Before you start

Complete Frontend Development with TypeScript first. You should already understand HTML, browser modules, DOM elements, npm packages, and strict TypeScript.

Install Node.js 24 LTS. Open a new terminal and verify it:

```text
node --version
npm --version
```

The Node.js result should start with `v24.`. If either command is missing, install Node.js from its official website and reopen the terminal.

## What you will learn

By the end of this module, you will be able to:

- distinguish the Vue playground from a local project
- create a Vue and TypeScript project with the official generator
- make deliberate generator choices instead of accepting unexplained features
- identify the HTML root, TypeScript entry file, and first component
- run development, type-checking, and production-build commands

## 1. Choose online or local practice

The official Vue playground is useful for trying a component without installation. It hides project creation, dependency installation, and build output, so it does not replace local setup. Complete the local steps for this course.

In a folder used for disposable practice, run:

```text
npm create vue@latest
```

This command downloads and runs `create-vue`, the official Vue project generator.

Choose these answers:

```text
Project name: vue-course
Add TypeScript? Yes
Add JSX support? No
Add Vue Router? No
Add Pinia? No
Add Vitest? No
Add an end-to-end testing solution? No
Add ESLint? Yes
Add Prettier? No
Add Vue DevTools? No
```

Router, Pinia, and testing are introduced in later modules. Adding them now would create files that have not been explained.

## 2. Install packages and run the project

The generator creates files but does not install packages for npm. Run:

```text
cd vue-course
npm install
npm run dev
```

Open the local address printed by Vite, commonly `http://localhost:5173`. Stop the server with `Ctrl+C`.

## 3. Read the generated files

The important starting structure is:

```text
vue-course/
  public/
  src/
    assets/
    components/
    App.vue
    main.ts
  env.d.ts
  eslint.config.ts
  index.html
  package-lock.json
  package.json
  tsconfig.app.json
  tsconfig.json
  vite.config.ts
```

- `index.html` contains the DOM element with `id="app"`.
- `src/main.ts` creates and mounts the Vue application.
- `src/App.vue` is the first Single-File Component.
- `env.d.ts` supplies types used by the Vite environment.
- `package.json` lists packages and scripts.
- `vite.config.ts` configures the development and build tool.
- `node_modules` contains installed packages and should not be edited or committed.
- `dist` is generated by a production build and should not be edited.

## 4. Follow the mount path

Open `index.html` and find:

```html
<div id="app"></div>
<script type="module" src="/src/main.ts"></script>
```

Replace `src/main.ts` with a version that reports a missing mount element clearly:

```ts
import "./assets/main.css";
import { createApp } from "vue";
import App from "./App.vue";

const root = document.querySelector("#app");

if (root === null) {
  throw new Error("Expected an element with id app");
}

createApp(App).mount(root);
```

`createApp(App)` creates an application whose root component is `App`. `mount(root)` connects that application to the real DOM element. The null check protects the runtime boundary between the HTML file and the TypeScript file.

## 5. Write the first component

Replace `src/App.vue` with:

```vue
<script setup lang="ts">
const courseName: string = "Vue Study Planner";
</script>

<template>
  <main>
    <h1>{{ courseName }}</h1>
    <p>My first Vue component is mounted.</p>
  </main>
</template>
```

A `.vue` file can contain script, template, and style sections. `lang="ts"` enables TypeScript in the script. The double braces place the string value into the rendered text.

## 6. Verify the project

Stop the server, then run:

```text
npm run lint
npm run type-check
npm run build
```

Type checking Vue templates needs Vue-aware tooling, which is why the generated project provides `type-check` in addition to ordinary TypeScript configuration.

## Common mistakes

### Running `vue create`

That command belongs to the older Vue CLI. New projects should use `npm create vue@latest`.

### Enabling every optional feature

More generated files do not produce more understanding. Add Router, Pinia, and test tools when their modules teach them.

### Omitting `lang="ts"`

Without it, the script section is treated as JavaScript rather than TypeScript.

### Editing generated folders

Change source files, not `node_modules` or `dist`.
