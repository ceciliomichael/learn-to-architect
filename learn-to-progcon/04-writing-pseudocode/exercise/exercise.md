# Module 04 Exercises

Complete these tasks with paper or a text editor. No programming language is required.

## Warm-up: Keyword fill-in

Rewrite the following casual notes as course-style pseudocode. Use `START`, `END`, `INPUT`, `SET`, and `OUTPUT`. Indent the body.

Casual notes:

```text
get the hours worked
get the hourly pay rate
pay is hours times rate
show the pay
```

Then desk-check with hours `10` and rate `15`. State the expected output.

## Guided exercise: Build a design in order

Problem:

```text
Read three item prices. Calculate the sum of the three prices. Show the sum.
```

Follow these steps:

1. List the inputs and the output in plain words.
2. Write pseudocode with `START` and `END`.
3. Use readable names such as `price1`, `price2`, `price3`, and `sum`.
4. Indent all lines between `START` and `END`.
5. Desk-check with prices `5`, `10`, and `2.5`. Show the expected sum.

## Independent exercise: Two sequential designs

Write full pseudocode for both problems. Each design must include `START`, `END`, readable names, indentation, and only sequential steps.

**Problem A: Rectangle area**

```text
Read width and height. Show the area (width times height).
```

**Problem B: Class average message values only (sequence form)**

```text
Read two quiz scores. Calculate the average. Show the average.
```

Requirements:

1. Pseudocode for A.
2. Pseudocode for B.
3. Desk-check A with width `4` and height `3`.
4. Desk-check B with scores `88` and `92`.
5. In two sentences, explain why these designs are pseudocode rather than a full programming language.

## Debugging / desk-check task

A learner submitted this pseudocode for "total = price * quantity, then show total":

```text
begin
input x
input y
total = x times y somehow
print it
```

1. List at least four problems based on this module (keywords, names, structure, precision, START/END style, indentation).
2. Rewrite correct course-style pseudocode.
3. Desk-check your rewrite with price `6` and quantity `7`. State expected output.
4. Optional alternative: rewrite the same design using `BEGIN`/`END` and `DISPLAY` instead of `START`/`END` and `OUTPUT`, still with clear names.
