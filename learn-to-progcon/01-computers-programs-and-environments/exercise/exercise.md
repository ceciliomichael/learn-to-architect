# Module 01 Exercises

Complete these tasks with paper or a text editor. You do not need a programming language for this module.

## Warm-up: Label a familiar task

Pick one everyday computer task from this list (or choose a similar one):

- looking up a product price online
- checking attendance in a class app
- converting a recipe temperature
- sending a short message

Write four short labels for that task:

1. **Input**: what information goes in
2. **Processing**: what work is done
3. **Output**: what comes out
4. **Storage**: what is saved (if anything)

Keep each label to one or two sentences.

## Guided exercise: Hardware, software, program

A school office uses a computer to print a student schedule.

Follow these steps:

1. List three pieces of **hardware** that might be involved.
2. List two pieces of **software** that might be involved (system or application).
3. In one or two sentences, describe the **program's** job (what instructions it is carrying out for the office worker).
4. Give one example of **source instructions** (write three plain steps, not a real programming language).
5. Give one example of a **result** the office worker or student would see or receive.

## Independent exercise: Two environments

You are designing a simple shopping total helper.

Requirements:

1. Describe the **programming environment** for the person who will write and test the instructions (tools, what they see, what they care about).
2. Describe the **user environment** for the shopper or cashier who only needs a correct total (what they see, what they care about).
3. Write a table or two short bullet lists that compare those environments on these points:
   - main person
   - what is visible
   - main goal
4. Invent one mistake that could happen in the programming environment (for example, a wrong formula written in the instructions).
5. Invent one problem that could appear only in the user environment (for example, a hard-to-read screen), even if the math is correct.

## Debugging / desk-check task

A learner wrote this description of a cafe order system. Find and correct the mistakes in the description. Rewrite the corrected version.

```text
The keyboard is software because it helps the user type.
The order program is hardware because it sits inside the computer.
Source instructions and results are the same thing: whatever appears on the screen.
Storage is optional because the total only needs to exist for one second.
A programming environment is where customers pay for coffee.
```

For each original sentence, state what was wrong and write a better sentence.

## Completion checklist

- [ ] I labeled input, processing, output, and storage for a real task.
- [ ] I can give examples of hardware and software.
- [ ] I described a program as a set of instructions, not as magic.
- [ ] I separated source instructions from results.
- [ ] I compared a programming environment with a user environment.
- [ ] I corrected the mistaken cafe description.
