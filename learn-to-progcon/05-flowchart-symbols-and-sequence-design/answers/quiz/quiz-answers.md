# Module 05 Quiz Answers

## Answer 1

- `END` : terminal
- `average = sum / count` : process
- `Enter student name` : input/output
- `score >= 60 ?` : decision

The shape signals the kind of step. The label supplies the details.

## Answer 2

They describe the same algorithm. If the order differs, one design is wrong or incomplete. Matching order keeps the plan consistent for people who read either form and for later coding.

## Answer 3

`Enter price` is an input action, not a calculation or internal update. It should use an input/output parallelogram, not a process rectangle.

## Answer 4

Displayed value: `32`.

Process step:

- `fahrenheit = (0 * 9 / 5) + 32`
- `fahrenheit = 0 + 32`
- `fahrenheit = 32`

Then the chart displays `32`.

## Answer 5

Good habits (any two such as these):

- list ordered steps before drawing
- leave space for START at the top and END at the bottom
- use short labels and large enough symbols
- walk the finished chart with your finger against the pseudocode

Common layout mistake to avoid (any one such as these):

- crossing arrows when a straight vertical path would work
- overcrowding symbols so labels are unreadable
- missing arrows between steps
- putting several unrelated actions into one vague box
