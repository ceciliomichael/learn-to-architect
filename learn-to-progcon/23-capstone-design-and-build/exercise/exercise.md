# Module 23 Exercises

This module is project work. Complete every deliverable for one chosen option (A, B, or C from the lesson).

## Warm-up: Scope your project

1. Choose Option A, B, or C.
2. Write a one-sentence problem statement.
3. List inputs, processes, and outputs in a short bullet list.
4. Cross out any feature that is not required for a minimum working version.

## Guided exercise: Design pack

Create these artifacts for your chosen project:

1. Numbered algorithm (at least 8 clear steps, or fewer if modules make steps shorter but still complete).
2. Pseudocode with a main module and at least two helper modules/functions.
3. Hierarchy chart in plain text.
4. Flowchart for the main path (text diagram is fine).
5. Trace table for one normal sample.

Follow this order. Do not jump to Python until the design pack exists.

## Independent exercise: Build and test

1. Implement the design in one Python file.
2. Run the normal sample from your trace table.
3. Run one boundary case and one edge case.
4. Fill a small test log:

| Case | Inputs | Expected | Actual | Pass? |
| --- | --- | --- | --- | --- |
| Normal |  |  |  |  |
| Boundary |  |  |  |  |
| Edge |  |  |  |  |

5. Write the short presentation note described in the lesson.

## Debugging / review task

Review your own project with this checklist and fix anything missing:

1. Every function has one clear job.
2. Every list index used is valid for the logical size.
3. Every loop can terminate.
4. Every selection branch needed by the requirements exists.
5. Program output matches the requirements list, not only your memory of the idea.

## Completion checklist

- [ ] I chose one scoped project.
- [ ] I wrote requirements (IPO and constraints).
- [ ] I produced algorithm, pseudocode, hierarchy chart, flowchart, and a trace.
- [ ] I implemented the design in Python.
- [ ] I tested normal, boundary, and edge cases.
- [ ] I wrote a short presentation note.
