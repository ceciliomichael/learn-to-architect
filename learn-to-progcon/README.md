# Learn Programming Concepts (ProgCon) Step by Step

Welcome. This course is for you even if you have never written code before.

ProgCon stands for Programming Concepts. You will learn how to understand a problem, design a solution with clear steps, check that logic by hand, and only later turn the design into Python. Flowcharts, pseudocode, conditions, loops, arrays, and modular design are the heart of the course.

You do not need a programming language on day one. You need careful thinking, a pencil or text editor, and the habit of checking each step.

## What ProgCon is

Programming is more than typing symbols into a computer. Before a program can run well, someone must:

1. understand the problem
2. invent a method that solves it
3. write that method precisely enough that another person or a machine can follow it
4. check the method for mistakes
5. implement and test it

This course teaches those ideas in order. You will design with pseudocode and flowcharts first. Python appears in the final part so you can see how a finished design becomes running code.

This course is not a complete professional Python path. After ProgCon, you can continue with [`learn-to-python/`](../learn-to-python/README.md) for deeper language study.

## How to use this course

Work through the modules in number order. For each module:

1. Read its `README.md` slowly.
2. Draw or type every example yourself. Do not only skim diagrams.
3. Pause when the lesson asks you to predict a result or complete a trace.
4. Complete `exercise/exercise.md` without opening the answers.
5. Check your designs and, later, your Python output against the requirements.
6. Complete `quiz/quiz.md` without looking ahead.
7. Read both answer files, including their explanations.
8. Change one exercise solution to see what happens.

It is normal to repeat a module. Learning design takes practice, not speed.

## Tools you will use

### For design work (Modules 01 to 19)

You need:

- paper and a pen or pencil, or a simple drawing app
- a plain text editor for pseudocode

You do not need a compiler for the early modules. The skill is writing and checking logic.

### For Python work (Modules 20 to 23)

#### Option 1: Use an online environment

Online environments such as [Programiz Python Compiler](https://www.programiz.com/python-programming/online-compiler/) or [Replit](https://replit.com/languages/python3) run in a web browser and require no setup. This is the simplest choice when you first reach Module 20.

#### Option 2: Work on your computer

Install a current Python release from <https://www.python.org>. Course examples target Python 3.12 and newer.

Open a terminal and check the installation:

```text
python --version
```

On some Windows systems, this command works instead:

```text
py --version
```

Create a file named `main.py`, then run it with:

```text
python main.py
```

## What to do when a design or program fails

Use this routine:

1. Re-read the problem statement carefully.
2. Compare your steps with the required order of operations.
3. For designs, build a trace table with sample values.
4. For Python, read the first error message and the line it mentions.
5. Check spelling, letter case, and indentation.
6. Make one small change, then check again.

Do not rewrite everything at once. A small change makes its effect easier to understand.

## Suggested pace

For Modules 01 to 10, one module every two or three days is a comfortable pace. Selection, loop, and array modules may need a few extra practice sessions. The Python modules and the capstone often need a week each. A useful study session is 30 to 60 minutes followed by a break.
