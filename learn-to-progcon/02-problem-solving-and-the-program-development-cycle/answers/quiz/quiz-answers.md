# Module 02 Quiz Answers

## Answer 1

1. Understand the problem  
2. Plan the logic  
3. Write the design or code  
4. Test  
5. Maintain and document  

## Answer 2

At least one of these is missing (name any essential gap):

- the price of one notebook, or a way to input that price
- the number of notebooks (quantity)
- whether tax or discounts apply
- what exactly to show as the result

A complete plan needs enough inputs and rules to compute a definite total.

## Answer 3

**B.** Identify inputs, describe processing, identify outputs.

IPO frames the problem as data in, work done, and results out.

## Answer 4

Rework becomes likely when wrong averages are discovered and the design or code must be fixed under pressure. The skipped step is **testing** (desk checking the plan with sample scores would have shown that dividing by 2 is wrong for three scores).

You could also note that understanding or planning was weak if "three scores" was never reflected in the divisor. The direct failure described in the question is the missing test that would catch divide-by-2.

## Answer 5

Development is a cycle because later discoveries send you back to earlier steps. Testing can reveal a need to replan. New user needs return you to understanding. Maintenance continues after the first working version. Work is repeated and refined, not finished in one irreversible pass.
